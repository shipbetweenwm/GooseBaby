# 鹅宝架构与智能体能力优化建议

## 一、当前架构分析

### 1.1 核心架构

```
┌─────────────────────────────────────────────────────────┐
│                    用户交互层                              │
│  (PetWindow + ChatPanel + SettingsPanel)                 │
└─────────────────────┬───────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────┐
│                   AgentLoop                               │
│  (Claude Code 风格循环，支持5种模式)                        │
│  ├─ Ask: 只回答不操作                                      │
│  ├─ Craft: 立即执行                                        │
│  ├─ Plan: 先规划后执行                                     │
│  ├─ Team: 多Agent协作                                      │
│  └─ CUA: 操控电脑                                          │
└─────────────────────┬───────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────┐
│                   技能层 (Skills)                          │
│  (AgentSkill + CUA + SubAgentSkill + ...)                │
└─────────────────────┬───────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────┐
│                  PetEngine                                │
│  (状态管理 + 行为状态机 + 属性衰减 + 金币挂机)               │
└─────────────────────────────────────────────────────────┘
```

### 1.2 当前优势

**架构优势**:
- ✅ **模块化设计**: AgentLoop、PetEngine、Skills分层清晰
- ✅ **多模式支持**: 5种Agent模式满足不同场景
- ✅ **Hook系统**: 生命周期钩子，灵活扩展
- ✅ **取消机制**: 支持中断长时间任务
- ✅ **循环防护**: 检测重复调用、停滞轮数

**智能体能力**:
- ✅ **CUA进程内架构**: CGEvent FFI,性能高
- ✅ **多Agent协作**: Team模式支持并行
- ✅ **计划确认**: Plan模式可规划复杂任务
- ✅ **记忆系统**: 长期记忆+向量检索

## 二、架构优化空间

### 2.1 缺少统一的状态管理框架

**当前问题**:
- PetEngine管理宠物状态，但缺少全局应用状态管理
- AgentLoop的上下文(ctx)是临时对象，无法跨会话持久化
- 各模块状态分散，难以统一管理和恢复

**优化方案**: 引入全局状态管理框架

```dart
/// 全局应用状态管理器
class AppStateManager extends ChangeNotifier {
  // ── 会话状态 ──
  SessionState _session;
  
  // ── 用户状态 ──
  UserProfile _user;
  
  // ── 宠物状态 ──
  PetState _pet;
  
  // ── Agent状态 ──
  AgentState _agent;
  
  // ── 设置状态 ──
  SettingsState _settings;
  
  // ── 状态持久化 ──
  Future<void> save();
  Future<void> restore();
  
  // ── 状态快照(用于撤销) ──
  AppStateSnapshot snapshot();
  void rollback(AppStateSnapshot snapshot);
}

/// 应用状态快照
class AppStateSnapshot {
  final DateTime timestamp;
  final SessionState session;
  final PetState pet;
  final AgentState agent;
  
  // 用于实现"时间回溯"功能
}
```

**优势**:
- 统一的状态管理入口
- 支持状态持久化和恢复
- 支持撤销/回滚操作
- 方便调试和监控

### 2.2 AgentLoop缺少异步任务调度

**当前问题**:
- 所有任务都是同步执行，用户必须等待
- 无法在后台运行长时间任务(如文件整理、数据分析)
- 缺少任务队列和优先级机制

**优化方案**: 引入任务调度器

```dart
/// 后台任务调度器
class TaskScheduler {
  // ── 任务队列 ──
  final List<ScheduledTask> _queue = [];
  
  // ── 执行中的任务 ──
  final Map<String, RunningTask> _running = {};
  
  // ── 已完成的任务 ──
  final List<CompletedTask> _completed = [];
  
  /// 提交后台任务
  Future<String> submit(
    String taskName,
    Future<void> Function() task, {
    TaskPriority priority = TaskPriority.normal,
    DateTime? scheduledAt,
  });
  
  /// 查询任务状态
  TaskStatus status(String taskId);
  
  /// 取消任务
  void cancel(String taskId);
  
  /// 暂停/恢复任务
  void pause(String taskId);
  void resume(String taskId);
}

enum TaskPriority {
  low,      // 低优先级(如文件整理)
  normal,   // 正常优先级
  high,     // 高优先级(如紧急提醒)
  realtime, // 实时优先级(如用户正在等待的任务)
}

/// 使用示例
// 用户: "帮我整理一下下载文件夹"
// 宠物: "好的~我会在后台帮你整理,完成后告诉你哦!"
scheduler.submit(
  '整理下载文件夹',
  () async {
    // 整理逻辑...
  },
  priority: TaskPriority.low,
);
```

**优势**:
- 支持后台执行长时间任务
- 用户无需等待，提升体验
- 支持任务优先级和调度
- 支持任务进度查询

### 2.4 缺少分布式Agent架构

**当前问题**:
- Team模式的多个Agent在同一进程内运行
- 无法跨设备协作(如手机+电脑)
- 无法利用云端算力

**优化方案**: 分布式Agent架构

```dart
/// Agent节点
class AgentNode {
  /// 节点ID
  final String id;
  
  /// 节点类型
  final AgentNodeType type;
  
  /// 节点能力
  final List<String> capabilities;
  
  /// 通信通道
  final AgentChannel channel;
  
  /// 执行任务
  Future<AgentResult> execute(AgentTask task);
}

/// Agent节点类型
enum AgentNodeType {
  local,   // 本地节点(当前设备)
  remote,  // 远程节点(其他设备)
  cloud,   // 云端节点
}

/// Agent集群管理器
class AgentCluster {
  final List<AgentNode> _nodes = [];
  
  /// 注册节点
  void register(AgentNode node);
  
  /// 任务分发
  Future<AgentResult> dispatch(AgentTask task) async {
    // 1. 分析任务需求
    final requirements = analyzeTask(task);
    
    // 2. 选择最佳节点(考虑能力、负载、延迟)
    final node = selectNode(requirements);
    
    // 3. 执行任务
    return await node.execute(task);
  }
  
  /// 负载均衡
  AgentNode selectNode(TaskRequirements req);
}
```

**使用场景**:
```
用户: "帮我把手机里的照片整理一下"
宠物: "好的~我会让手机端的鹅宝帮你整理,整理好了告诉你!"
(cluster.dispatch(task) → 手机节点执行)
```

**优势**:
- 跨设备协作
- 云端算力扩展
- 负载均衡
- 容错性(节点故障可切换)

### 2.5 缺少多模态融合架构

**当前问题**:
- CUA只支持视觉(截图)，不支持语音、触控等
- 缺少多模态信息的融合机制
- 无法处理"听语音+看屏幕+理解上下文"的综合场景

**优化方案**: 多模态融合架构

```dart
/// 多模态感知器
class MultimodalPerception {
  // ── 视觉感知 ──
  VisualPerception visual;
  
  // ── 听觉感知 ──
  AuditoryPerception auditory;
  
  // ── 触控感知 ──
  TactilePerception tactile;

  Future<MultimodalContext> perceive() async {

    final futures = await Future.wait([
      visual.capture(),
      auditory.listen(),
      tactile.sense(),
    ]);
    
    // 融合多模态信息
    return _fuse(futures);
  }
}

```

**使用场景**:
```
用户: (看屏幕+叹气+说"唉...")
宠物: (多模态感知 → 视觉:看到失败提示 + 听觉:叹气 + 语音:"唉")
宠物: "看起来遇到了挫折...想跟我说说发生了什么吗?"
```

**优势**:
- 更准确的情感理解
- 更丰富的交互方式
- 更自然的用户体验

```dart
/// 情感推理引擎
class EmotionReasoningEngine {
```

### 3.3 缺少任务规划和推理能力

**当前问题**:

**优化方案**: 任务推理引擎

```dart
/// 任务推理引擎
class TaskReasoningEngine {
  /// 任务可行性分析
  Future<FeasibilityReport> analyzeFeasibility(AgentTask task) async {
    // 1. 任务分解
    final subtasks = _decompose(task);
    
    // 2. 依赖关系分析
    final dependencies = _analyzeDependencies(subtasks);
    
    // 3. 风险评估
    final risks = _assessRisks(subtasks);
    
    // 4. 资源需求分析
    final resources = _analyzeResources(subtasks);
    
    return FeasibilityReport(
      subtasks: subtasks,
      dependencies: dependencies,
      risks: risks,
      resources: resources,
      isFeasible: risks.every((r) => r.level != RiskLevel.critical),
    );
  }
  
  /// 任务优化建议
  Future<OptimizationPlan> optimize(AgentTask task);
}

/// 可行性报告
class FeasibilityReport {
  /// 子任务列表
  final List<Subtask> subtasks;
  
  /// 依赖关系图
  final DependencyGraph dependencies;
  
  /// 风险列表
  final List<Risk> risks;
  
  /// 资源需求
  final ResourceRequirements resources;
  
  /// 是否可行
  final bool isFeasible;
  
  /// 不可行原因
  final String? infeasibleReason;
}

/// 风险
class Risk {
  final String description;
  final RiskLevel level;
  final String mitigation;
}

enum RiskLevel {
  low,      // 低风险
  medium,   // 中风险
  high,     // 高风险
  critical, // 严重风险(不可行)
}
```

**使用场景**:

**优势**:

```dart
/// 工具使用学习器
class ToolLearningEngine {

    return ToolStrategy(
      bestPractices: successPatterns,
      pitfalls: failurePatterns,
      suggestedApproach: _suggestApproach(successPatterns, failurePatterns),
    );
  }
  
  /// 智能工具选择
  Future<ToolSelection> selectTool(AgentTask task) async {
    // 根据任务和过往经验选择最佳工具
  }
}

/// 工具使用记录
class ToolUseRecord {
  final String toolName;
  final Map<String, dynamic> arguments;
  final bool isSuccess;
  final String? errorMessage;
  final int executionTime;
  final UserFeedback? feedback;
}
```

**使用场景**:
```

- 第1次使用file_write失败: 参数错误,缺少file_path

```

**优势**:
- 从经验中学习
- 避免重复犯错
- 优化工具使用策略
- 提升任务成功率
